# Misc. Utility Functions [(API)](https://public.msli.com/lcs/muscle/html/group__misutilityfunctions.html)

```#include "util/MiscUtilityFunctions.h"```

Various functions that didn't fit well anywhere else are kept here.

* [ParseArgs()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga94e8b5be40b8608421d42287c5f8c466) and [ParseFile()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gab1269617bc0cf5c68ccff49f51f90d8c) parse a textual list of arguments (of the form `key` or `key=value`) into a [Message](https://public.msli.com/lcs/muscle/html/classmuscle_1_1Message.html) object for easy argument-parsing.  The input can be in the form of a single string or a file.
* [UnparseArgs()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gaf5b9da4837a9e8600ff052e409557d04) and [UnparseFile()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gae44eb3be9a653e1028dbe5f440a9fbf0) do the opposite -- converts a [Message](https://public.msli.com/lcs/muscle/html/classmuscle_1_1Message.html) object's string fields back into a text-string that can be written to disk as a config file.
* [ParseConnectArg()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gab45a86e8947edc0e3d7d93f18c2c4cac) takes a [String](https://public.msli.com/lcs/muscle/html/classmuscle_1_1String.html) of the form "hostname:port" and parses out the hostname and port into separate fields.
* [HandleStandardDaemonArgs()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga3fdfd5f3807823e704e036ff677d3bec) takes a [Message](https://public.msli.com/lcs/muscle/html/classmuscle_1_1Message.html) of command line / config arguments and looks for common arguments to handle in the standard way (e.g. "displaylevel=debug", "console", "nice", etc)
* [Crash()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gac3d0b17eff41953d8c04c7822dfbca2c) does what the name suggests it will do.
* [BecomeDaemonProcess()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga98513276cd016d020f48ae8cdd7139ee) transfers the current process into the background, detached it from its parent process/shell.
* [SpawnDaemonProcess()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gaa6d8a9eacb3a7847f8198811a3afe130) spawns a child process that will run in the background as a fork()'d copy of the current process.
* [NybbleizeData()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga18ff6c9b8cfebb1b3980e1034a645748) converts binary data into an ASCII-compatible 4-bits-per-character "safe" string, for passing the data over non-binary-safe transports.  [DenybbleizeString()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gadab83219ca9bb98ff8f5c3f9a5ea1f70) does the opposite.
* [MemMem()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga1c0fa33d5a6ffe65f97fa1efec670e91) looks for the first occurrence small binary sequence within a larger one.  (Like `strstr()`, except for raw binary)
* [PrintHexBytes()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga6a80ecdf6a4ae754f15d2209a75437b3) prints out the specified binary data, in a pretty hexadecimal format, annotated with an ASCII sidebar (useful for debugging)
* [LogHexBytes()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga35c294189ad5d5fbadfa211c15e5d403) does the same thing as [PrintHexBytes()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga6a80ecdf6a4ae754f15d2209a75437b3), except the output goes to the log instead of just to stdout.
* [HexBytesToAnnotatedString()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gaf9ac97b2d58bd4fb940f61493f847183) does the same thing again, except the output is returned as a [String](https://public.msli.com/lcs/muscle/html/classmuscle_1_1String.html) instead of output anywhere.
* [ParseHexBytes()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga7db1efc3c6ec8779d6244ee1c6837f97) reads a human-readable string containing hex printouts (e.g. "1f 7e 26 35") and returns the corresponding binary data as a [ByteBufferRef](https://public.msli.com/lcs/muscle/html/classmuscle_1_1ByteBuffer.html).
* [HexBytesToString()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga327c28882d7733f28c353e7aee52bb3a) converts a raw binary sequence into a human-readable string (e.g. "1f 7e 26 35"), without any annotation.
* [FileExists()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gad8587f097f1c4d0c2c019a162d7e79bb), [RenameFile()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga0fe4cda3f9eb919825f04112c898f786), [CopyFile()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gaf9f3598f109063f1d1069d891b2ee5d8), and [DeleteFile()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#gac5fa510eb65b6430084681cf110abc00) all do what their names suggest.
* `Win32AllocateStdioConsole()` opens a DOS prompt so that stdout output can be viewed (under Windows only; other platforms don't need this)
* [GetSystemMemoryUsagePercentage()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga0579e7b0e2ef852e89c695b983db10b2) returns the percentage of the host computer's RAM that is currently in use.
* [GetProcessMemoryUsage()](https://public.msli.com/lcs/muscle/html/group__miscutilityfunctions.html#ga35a3993070d24b3c98b2242c8ba83542) returns the (approximate) amount of RAM being used by the current process, in bytes.
